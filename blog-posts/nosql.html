Overview

Most people are familiar with SQL and have a basic understanding of the various RDBMS out there - Postgres, MySQL, Oracle etc. However recently a newer type of database is becoming popular - NoSQL such as MongoDB, Redis, CoudchDb etc. Taken from Wikipedia NoSQL is defined as:

"A NoSQL (often interpreted as Not Only SQL) database provides a mechanism for storage and retrieval of data that is modeled in means other than the tabular relations used in relational databases."

SQL vs NoSQL

A brief comparison of the two types:

  • SQL databases are table based, NoSQL databases are document based
  • To query and interact with a SQL db we use SQL (structured query language), to interact with NoSQL we use database-specific syntax.
  • SQL dbs have predefined schemas NoSQL dbs do not require them.
  • NoSQL can provide benefits over SQL when used for big data in terms of speed and performance

Conclusion

Any topic concerning technology A vs technology B will always inspire heated debate about which is better. As with most technologies the choice of which type to use is dependent on needs and situation. The options given here offer different advantages and developers should think carefully about which choice is best for their needs.
